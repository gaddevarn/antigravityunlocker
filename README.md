# Antigravity Unlocker для Linux

Этот проект был адаптирован под запуск на Linux и под конкретный сценарий с локальной папкой Antigravity IDE. Он позволяет попробовать патч приложения в среде Linux, в том числе для папок вида `/home/USER/AntigravityIDE`.

## Что изменилось по сравнению с исходной версией

В этой версии были внесены следующие изменения:

- добавлена поддержка запуска на Linux;
- добавлен явный способ указать путь к установке Antigravity IDE через `--install-path` или `ANTIGRAVITY_INSTALL_PATH`;
- улучшена работа с папками, которые не лежат в стандартных местах Linux;
- добавлена поддержка пропуска этапа лицензии для локальных тестов через `AG_SKIP_LICENSE=1`;
- добавлена поддержка чтения секретного значения из `.secrets.json` или `.license_secret`, чтобы ключи были стабильнее между сборками;
- сохранён механизм патча файлов приложения и бинарников, но теперь он лучше подходит для сценария с локальной папкой и ручным путём.

## Важно

Этот проект всё ещё является экспериментальной адаптацией под Linux. Он может работать для вашей системы, но возможны следующие ограничения:

- не все сборки Antigravity IDE одинаковы и могут иметь отличающуюся структуру файлов;
- некоторые версии приложения могут требовать другие пути к ресурсам или другие файлы для патча;
- на некоторых системах нужны права администратора или доступ на запись в папку установки;
- ключи могут зависеть от секрета сборки, поэтому при повторной сборке они могут перестать работать, если не сохранить тот же секрет.

## Где скачать и как использовать

Для разблокировки обычно нужен лицензионный ключ. Его можно получить у автора/в источнике, который вы использовали для этой версии.

## Установка зависимостей

На Ubuntu / Linux Mint / Debian-подобных системах:

```bash
sudo apt-get update
sudo apt-get install -y cargo rustc
```

Если у вас другая система, установите Rust через официальный способ для вашей дистрибуции:
https://www.rust-lang.org/tools/install

## Сборка

Из корня проекта выполните:

```bash
cargo build --release
```

После сборки исполняемый файл будет доступен по пути:

```bash
./target/release/ag_unlocker
```

## Запуск на Linux

### Базовый запуск

```bash
./target/release/ag_unlocker
```

### Запуск с явным путём к папке Antigravity IDE

Если Antigravity IDE стоит не в стандартной папке, запускайте так:

```bash
./target/release/ag_unlocker --install-path /home/USER/AntigravityIDE
```

Или через переменную окружения:

```bash
export ANTIGRAVITY_INSTALL_PATH=/home/USER/AntigravityIDE
./target/release/ag_unlocker
```

### Запуск без проверки лицензии для локальных тестов

Если нужно только проверить сам процесс патча и не застревать на ключе, можно использовать:

```bash
AG_SKIP_LICENSE=1 ./target/release/ag_unlocker --install-path /home/USER/AntigravityIDE
```

## Как должна выглядеть папка Antigravity IDE

Программа ищет установку, если папка содержит такие элементы:

- папку `resources/`;
- файл `app.asar` или папку `app/` внутри `resources/`;
- либо исполняемый файл `agy` или `agy.exe`.

Пример структуры:

```text
/home/USER/AntigravityIDE/
  antigravity-ide
  resources/
    app/
      ...
  bin/
```

Подходящий путь может быть таким:

- `/home/USER/AntigravityIDE`
- `/home/USER/.local/share/Antigravity`
- `/opt/antigravity`
- `/opt/Antigravity`

## Что делать, если программа не видит папку

Если программа пишет, что установки Antigravity не найдены, проверьте следующее:

1. Путь указан абсолютно, например:
   ```bash
   /home/michael/AntigravityIDE
   ```
2. Папка действительно существует:
   ```bash
   ls -la /home/michael/AntigravityIDE
   ```
3. Внутри есть нужные элементы:
   ```bash
   find /home/michael/AntigravityIDE -maxdepth 2 | head -50
   ```
4. Если папка лежит в домашней директории, передавайте путь вручную:
   ```bash
   ./target/release/ag_unlocker --install-path /home/michael/AntigravityIDE
   ```

## Что делать, если нужны права администратора

На Linux запуск от администратора обычно делается через `sudo`:

```bash
sudo AG_SKIP_LICENSE=1 ./target/release/ag_unlocker --install-path /home/USER/AntigravityIDE
```

Если программа сообщает, что папка недоступна для записи, это обычно значит, что у текущего пользователя нет прав на изменение файлов внутри установки. В таком случае попробуйте запустить программу от имени администратора.

## Как работают ключи и секреты

Ключи привязаны к секрету сборки. Поэтому:

- ключ, который подошёл для одной сборки, может не подойти для другой;
- если вы пересобрали программу, старые ключи могут перестать работать;
- если нужен тот же ключ, нужно использовать тот же секрет, который был использован при его генерации.

Для удобства программа может читать секрет из:

- переменной окружения `AG_LICENSE_SECRET`;
- переменной окружения `LICENSE_SECRET`;
- файла `.secrets.json`;
- файла `.license_secret`.

Пример:

```bash
AG_LICENSE_SECRET='YOUR_SECRET' ./target/release/ag_unlocker
```

## Что ожидается после успешного запуска

Если всё настроено правильно, программа должна:

- принять ключ или пройти через `AG_SKIP_LICENSE=1`;
- обнаружить папку Antigravity IDE по переданному пути;
- выполнить патч и сообщить, что установка успешно обработана.

Пример успешного вывода:

```text
[OK] Успешно пропатчено: Antigravity IDE
```

## Ограничения и известные сложности

- на Linux некоторые версии приложения могут иметь другую структуру файлов, чем ожидалось;
- некоторые сборки могут не патчиться без прав администратора;
- если путь к установке неправильный, программа не сможет ничего изменить;
- если секрет сборки отличается, старые ключи могут не приниматься.

## Вопросы и ответы

### Работает ли это для Antigravity 2?
Да, в текущем сценарии это проверялось и работало для локальной папки Antigravity IDE.

### Почему на Windows и Linux поведение может отличаться?
Потому что Windows и Linux используют разные пути, разные разрешения и разные структуры запускаемых файлов.

### Почему антивирус видит программу как подозрительную?
Это обычное поведение для инструментов, которые изменяют исполняемые файлы и внутренние ресурсы приложения. Открытый исходный код помогает проверить, что делает программа, но при желании лучше использовать её в виртуальной машине или в изолированной среде.
